## 1. 게임 정체성

### 1.1 이것은 어떤 게임인가

**First Queen 4 HD Renewal**은 1994년 DOS 실시간 전술 RPG의 충실한 현대화 리메이크다.

| 이것은 | 이것이 아니다 |
|--------|-------------|
| 실시간 전술 RPG | 방치형 게임 |
| 플레이어가 직접 조작하는 액션 | 자동전투만 보는 관전 게임 |
| AI 부대원과 함께 싸우는 전투 | 편성만 하고 결과 기다리는 게임 |
| 스토리 기반 챕터 진행 | 스테이지 반복 파밍 |
| 원작 파라미터 100% 계승 | 밸런스 재설계 |

### 1.2 핵심 게임 루프

```
탐색 (맵 이동) → 전투 (Gocha-Kyara) → 휴식 (회복) → 이벤트/대화 → 챕터 진행
      ↑                                                              |
      +──────────────────────────────────────────────────────────────+
```

플레이어는 주인공 아레스를 **직접 WASD/조이스틱으로 조작**한다.
부대원(최대 7명)은 AI가 자동 제어하며, 플레이어는 **명령과 대형**으로 전술을 결정한다.

### 1.3 자동전투 = 편의 옵션

자동전투는 게임의 핵심이 아니라 **편의 기능**이다.

| 모드 | 설명 | 용도 |
|------|------|------|
| **수동 조작** (기본) | 주인공 직접 제어 + AI 부대원 | 메인 플레이 |
| **자동전투** (옵션) | 주인공도 AI 제어 | 레벨링/반복 전투 시 |
| **배속** (1x/2x) | 자동전투 시에만 가능 | 반복 전투 가속 |

자동전투는 "원하면 쓸 수 있다" 수준. 게임의 재미는 **직접 조작**에 있다.

## 2. 재미의 핵심: Gocha-Kyara 전투

### 2.1 왜 Gocha-Kyara가 재밌는가

```
일반 RPG:     내가 1명 제어 → 나머지는 메뉴 선택
Gocha-Kyara:  내가 1명 제어 → 나머지 7명이 AI로 자기 판단으로 전투
              → "야 거기 가지 마!" "잘했어!" "왜 쉬고 있어!"
              → 예측 불가능한 AI 행동이 만드는 전술적 혼돈의 재미
```

핵심 쾌감:
1. **나는 영웅이다**: 직접 조작하는 주인공의 액션 쾌감
2. **부대장이다**: AI 부대원에게 명령을 내리는 지휘의 쾌감
3. **혼돈의 전장**: 수십 유닛이 뒤엉켜 싸우는 난전의 박력
4. **전략적 판단**: 언제 돌격하고, 언제 후퇴할지 실시간 결정
5. **동료 관리**: 피로도/HP 관리, 유닛 전환, 부대 전환

### 2.2 레퍼런스 게임

| 게임 | 유사점 | 차이점 |
|------|--------|--------|
| **Unicorn Overlord** (2024) | 실시간 분대 이동, AI 행동 설정 | FQ4는 직접 1명 조작 |
| **Mount & Blade** | 직접 전투 + AI 부대 지휘 | FQ4는 2D, 더 많은 유닛 |
| **Pikmin** | 리더 1명 + 다수 추종자 | FQ4는 RPG 요소 강함 |
| **Dragon's Crown** | 실시간 액션 + AI 동료 | FQ4는 전술 요소 강함 |
| **Ogre Battle** | 부대 편성 + 실시간 이동 | FQ4는 직접 전투 참여 |

**POC-ACT-01: 조작 반응성 테스트**
```
검증: 입력 → 화면 반응 지연 측정
성공 기준:
  - 이동: 입력 후 50ms 이내 캐릭터 이동 시작
  - 공격: 버튼 후 100ms 이내 공격 모션 시작
  - 방향 전환: 즉각 반영 (30ms 이내)
측정: Flame update loop에서 input → position 변경까지 dt 측정
실패 시: 입력 버퍼링 또는 물리 연산 최적화
```

**POC-ACT-02: 공격 타격감 테스트**
```
검증: 공격 시 "때리는 느낌"이 나는가
(A) 기본: 데미지 숫자만 표시
(B) 중간: + 히트 플래시 + 넉백
(C) 풀: + 히트스톱(2~3프레임) + 카메라 미세 흔들림 + 타격음
성공 기준: (C) 버전에서 "때리는 느낌이 확실히 난다"
```

## 5. B. 부대 지휘 재미 (CMD: 28개)

> "나는 단순한 전사가 아니라 부대장이다."

### B1. 대형 시스템

| ID | 요소 | 설명 | POC 필요 |
|----|------|------|---------|
| CMD-01 | **V자 대형** | 리더 앞, 부대원 뒤쪽 V자 배치 → 기본 돌파 | Yes |

| CMD-03 | **CIRCLE 원형** | 리더 보호 → 수비/포위 | Yes |

| CMD-07 | **대형 유지력** | 이동 중에도 대형 유지 | No |
| CMD-08 | **대형 시각 표시** | 현재 대형 아이콘 + 예상 위치 표시 | No |

**POC-CMD-01: 대형 전투 영향 테스트**
```
검증: 대형 변경이 전투 결과에 체감 가능한 영향을 주는가
시나리오:
  4명 아군 vs 4명 적, 플레이어 수동 조작
  (A) V자 대형으로 정면 돌파
  (B) 산개 대형으로 같은 적과 전투
  (C) 원형 대형으로 같은 적과 전투
측정: 전투 시간, 사상자 수, HP 소모량
성공 기준:
  - 대형별 전투 양상이 눈으로 구분 가능
  - V자: 빠른 격파, 높은 피해
  - 원형: 느린 격파, 낮은 피해
  - 산개: 범위 공격 회피, 개별 격파
```

| CMD-13 | **전원 휴식 (REST)** | 제자리에서 피로 회복 | Yes |
| CMD-14 | **분산 (SPREAD)** | 산개 | No |
| CMD-15 | **명령 즉시 반응** | 명령 후 0.5초 이내 행동 변경 | Yes |
| CMD-16 | **명령 아이콘** | 현재 명령 상태 표시 | No |

**POC-CMD-02: 명령 체감 테스트**
```
검증: 명령이 즉각적이고 체감 가능한가
시나리오: 전투 중 FOLLOW → ATTACK_ALL → RETREAT 명령 순환
성공 기준:
  - ATTACK_ALL: AI가 적극적으로 전진/교전 (FOLLOW 대비 확연한 차이)
  - RETREAT: 전원 즉시 후방 이동 (교전 중이던 유닛도)
  - HOLD: 유닛이 지정 위치에서 이동하지 않음
  - 명령 전환 시 AI 행동 변화가 0.5초 이내 관찰 가능
```

## 14. PRD-0003 폐기 사유

PRD-0003 "천지창조"는 방치형 게임으로의 전환을 제안했으나, 이는 FQ4의 핵심 정체성과 충돌한다.

| PRD-0003 (폐기) | 본 GDD (채택) |
|-----------------|-------------|
| 전원 AI 자동전투 | 주인공 직접 조작 + AI 부대 |
| 방치 진행 | 실시간 탐색 + 전투 |
| 세로 모드 | 가로 모드 기본 |
| 스테이지 반복 파밍 | 스토리 기반 챕터 진행 |
| 오프라인 보상 | 없음 (순수 게임) |
| 소탕/스킵 | 없음 (직접 플레이) |
