# First Queen 4 HD Remake - 게임 설계 문서

**버전**: 1.0.0
**마지막 업데이트**: 2026년 2월
**개발 엔진**: Godot 4.4 (Forward+)
**타겟 해상도**: 1280x800 (원본 320x200의 4배)
**장르**: 실시간 전술 RPG
**플랫폼**: Windows (추후 멀티플랫폼)

---

## 1. 개요

### 1.1 게임 소개

**First Queen 4 HD Remake**는 1994년 Kure Software가 개발한 전설의 실시간 전술 RPG "First Queen 4"(DOS)를 현대의 Godot 4.4 엔진으로 완전히 리메이크하는 프로젝트입니다.

원작의 혁신적인 "Gocha-Kyara"(ごちゃキャラ) 시스템을 핵심으로 재현하면서, HD 그래픽, 향상된 UI, 그리고 다양한 게임플레이 개선사항을 제공합니다.

### 1.2 원작 정보

| 항목 | 내용 |
|------|------|
| **제목** | First Queen 4 (ファーストクイーン4) |
| **개발사** | Kure Software |
| **발매년도** | 1994년 |
| **플랫폼** | DOS (PC-9801) |
| **장르** | 실시간 전술 RPG |
| **특징** | Gocha-Kyara AI 자동 제어 시스템 |

### 1.3 리메이크의 비전

```
원작의 정신 보존 × 현대 게임플레이 개선 × HD 비주얼 업그레이드
```

- **역사적 충실성**: 원작의 스토리, 캐릭터, 게임 메커니즘 충실한 재현
- **시각적 개선**: 320x200 → 1280x800 고해상도, 향상된 색감, 애니메이션
- **인체공학**: 직관적 UI, 명확한 조작감, 접근성 개선
- **확장성**: 새로운 콘텐츠, 모드 지원, 커뮤니티 참여

### 1.4 타겟 플레이어

- **원작 팬**: First Queen 4의 향수와 재미를 추구하는 플레이어
- **전술 RPG 애호가**: 전략적 전투와 AI 시스템에 관심 있는 플레이어
- **고전 게임 애호가**: DOS 시대 게임의 매력을 인정하는 플레이어
- **한국 게이머**: 특히 1990년대 한국 게이머들의 추억의 게임

## 2. 핵심 시스템

### 2.1 Gocha-Kyara 자동 AI 제어 시스템

**Gocha-Kyara(ごちゃキャラ)**는 First Queen 4의 가장 혁신적인 게임플레이 요소로, 플레이어가 1명의 리더만 조작하고 나머지 부대원들은 AI가 자동으로 제어하는 시스템입니다.

#### 2.1.1 기본 개념

```
플레이어 조작 (리더) ←→ AI 자동 제어 (부대원)
```

**플레이어의 역할:**
- 1명의 주인공(리더) 직접 조작
- 부대 이동, 대형 변경, 전술 명령 내리기
- 전황 판단 및 전략적 결정

**AI의 역할:**
- 나머지 4명의 동료 자동 제어
- 리더 따라가기, 적 자동 감지
- 개별 성격에 맞는 행동
- 피로도/체력 기반 자동 판단

#### 2.1.2 AI 상태 머신

AI 유닛들은 9개의 상태를 가지는 상태 머신으로 행동합니다:

| 상태 | 설명 | 전환 조건 |
|------|------|---------|
| **IDLE** | 대기 | 리더 상실, 명령 대기 |
| **FOLLOW** | 리더 따라가기 | 기본 상태, 부대 유지 |
| **PATROL** | 순찰 | 리더 상실 시 자동 순찰 |
| **CHASE** | 적 추격 | 적 감지 시 자동 전환 |
| **ATTACK** | 공격 중 | 적 사거리 진입 |
| **RETREAT** | 후퇴 | HP 30% 이하 또는 피로도 70% 이상 |
| **DEFEND** | 방어 | 명령 또는 리더 근처 적 감지 |
| **SUPPORT** | 지원 | 추후 구현 (힐러/버퍼) |
| **REST** | 휴식 | 피로도 90% 이상 강제 |

#### 2.1.3 성격 시스템 (Personality)

각 부대원은 3가지 성격 중 하나를 가지며, 이는 AI 행동에 영향을 줍니다:

**AGGRESSIVE (공격적)**
- 진입 범위: 1.5배 확대
- 특징: 적극적으로 적 추격, 높은 공격력
- 위험도: 높음 (체력 소진 빠름)
- 적합 직업: 검사, 전사

**DEFENSIVE (방어적)**
- 진입 범위: 0.7배 축소
- 후퇴 체력 임계값: 1.3배 상향
- 특징: 리더 근처 유지, 신중한 전투
- 적합 직업: 기사, 탱커

**BALANCED (균형)**
- 표준 파라미터 사용
- 상황에 따라 유연하게 대응
- 위험도: 중간
- 적합 직업: 일반 검사, 궁수

**성격별 우선순위:**
```
공격적: 적 추격 > 리더 따라가기 (위험)
방어적: 리더 따라가기 > 적 추격 (안전)
균형:   상황에 따라 동적 결정
```

#### 2.1.4 대형 시스템 (Formation)

부대의 전술적 운영을 위해 5가지 대형을 제공합니다:

| 대형 | 설명 | 효과 |
|------|------|------|
| **V_SHAPE** | V자 대형 (기본) | 분산성 높음, 전투 강화 |
| **LINE** | 일렬 대형 | 빠른 이동, 통로 통과 용이 |
| **CIRCLE** | 원형 대형 | 방어력 우수, 모든 방향 커버 |
| **WEDGE** | 쐐기 대형 | 집중력 강화, 돌진 공격 |
| **SCATTERED** | 분산 대형 | 최대 분산, 광역 공격 회피 |

**부대 명령 시스템:**
- GATHER: 원형 대형으로 집합
- SCATTER: 분산 대형 전환
- ATTACK_ALL: 모든 부대원 공격 명령
- DEFEND_ALL: 모든 부대원 방어 명령
- RETREAT_ALL: 모든 부대원 후퇴 명령

### 2.2 피로도 시스템 (Fatigue System)

First Queen 4의 또 다른 혁신적 요소인 피로도 시스템은 모든 유닛의 지속성과 전술성을 결정합니다.

#### 2.2.1 피로도 메커니즘

**피로도 증가 요인:**
- **이동**: 10픽셀당 1 피로도
- **공격**: 1회당 10 피로도
- **스킬**: 사용 시 20 피로도
- **시간 경과**: 자연적 증가

**피로도 감소 요인:**
- **대기**: 초당 1 피로도 회복
- **휴식**: 초당 5 피로도 회복
- **특수 스킬**: 스킬 사용 시 일부 회복 가능

#### 2.2.2 피로도 단계별 페널티

**피로도 범위에 따른 상태 변화:**

| 피로도 | 상태 | 이동 속도 | 공격력 | 행동 |
|--------|------|---------|--------|------|
| 0-30% | NORMAL | 100% | 100% | ✅ 모든 행동 가능 |
| 31-60% | TIRED | 80% | 90% | ⚠️ 감속 상태 |
| 61-90% | EXHAUSTED | 50% | 70% | ⚠️ 심각한 감속 |
| 91-100% | COLLAPSED | 0% | 0% | ❌ 강제 휴식 |

#### 2.2.3 전술적 영향

피로도 시스템은 단순한 메커니즘을 넘어 게임의 전술성을 결정합니다:

**전략적 고려사항:**
- 장기전 대비 피로도 관리 필수
- 무리한 공격 → 피로도 급증 → 후퇴 강요
- 휴식 타이밍이 전국을 가르는 열쇠
- 부대원 교체를 통한 피로도 분산

**부대 운영 패턴:**
```
Phase 1: 신선한 부대 적극 공격 (피로도 0-30%)
Phase 2: 피로도 누적, 방어 위주로 전환 (30-60%)
Phase 3: 체력 2진 유닛 배치, 피로 부대원 교체 (60-90%)
Phase 4: 완전 휴식 후 재출격 (90% 이상)
```

### 2.3 전투 시스템 (Combat System)

#### 2.3.1 기본 전투 메커니즘

**공격 판정:**
- **명중률**: 95% 기본 (회피율 5%)
- **크리티컬**: 5% 확률, 2배 데미지
- **데미지 편차**: ±10% 분산

**최소 데미지:**
- 모든 공격은 최소 1 데미지 이상 보장
- 회피/실패 시 데미지 없음

#### 2.3.2 사거리 시스템

| 사거리 | 유닛 타입 | 사용 직업 |
|--------|---------|----------|
| **근거리 (근접)** | 1 칸 | 검사, 전사, 기사 |
| **중거리** | 2-3 칸 | 궁수, 마법사 |
| **원거리** | 4+ 칸 | 원거리 마법사 |

#### 2.3.3 전투 이벤트

```
전투 시작 → 명중 판정 → 데미지 계산 → 팝업 표시 → 피로도 증가
   ↓
명중/실패 → 크리티컬 판정 → 이펙트 재생 → 상태 변화
   ↓
사망 시 → 경험치 획득 → 부대 재정렬
```

### 2.4 경험치 및 성장 시스템 (Experience & Progression)

#### 2.4.1 경험치 획득

**경험치 획득 방법:**
- 적 유닛 처치: 처치된 유닛 경험치
- 전투 참여 보너스: 라운드당 기본치
- 임무 완료: 챕터별 추가 경험치

**경험치 분배:**
- 실제 공격을 한 유닛: 100% 획득
- 같은 부대 유닛: 20% 간접 경험치
- 전투 참여 유닛: 레벨별 차등 경험치

#### 2.4.2 레벨업 시스템

**능력치 상승 (레벨 업 시):**
- 최대 HP: +5~10
- 공격력: +1~2
- 방어력: +0~1
- 최대 피로도: +5~10

**직업별 성장 곡선:**
- 전사 타입: 체력 우선 성장
- 궁수 타입: 균형 성장
- 마법사 타입: 마력 우선 성장

#### 2.4.3 장비 시스템

**장비 종류:**
- 무기 (검, 창, 활, 지팡이)
- 방어구 (갑옷, 투구, 방패)
- 액세서리 (반지, 목걸이)

**장비 효과:**
- 공격력 증가
- 방어력 증가
- 특수 능력 부여
- 속성 저항 추가

## 7. 게임플레이 흐름

### 7.1 전투 루프

```
1. 플레이어 이동
   ├─ WASD로 주인공 이동
   └─ AI가 부대원 자동 조종 (Gocha-Kyara)

2. 전투 감지
   ├─ 적 감지 범위 내 진입
   └─ AI 자동 추격 시작

3. 공격
   ├─ Space 키 또는 우클릭 → 공격 명령
   ├─ 명중/회피/크리티컬 판정
   └─ 데미지 적용

4. 피로도 관리
   ├─ 이동 시 피로도 증가
   ├─ 공격 시 피로도 추가 증가
   └─ 피로도 높음 → 성능 저하 또는 강제 휴식

5. 회복/교체
   ├─ 휴식 → 피로도 회복
   ├─ 아이템 사용 → HP/상태 회복
   └─ 부대 교체 → 신선한 유닛 투입

6. 전투 종료
   ├─ 모든 적 격퇴 → CLEAR
   └─ 모든 아군 전사 → GAME OVER
```

### 7.2 챕터 진행

```
Chapter Start
    ↓
Tutorial / Story
    ↓
Main Battle
    ├─ Phase 1: 적 감지 및 첫 전투
    ├─ Phase 2: 피로도 누적 시작
    └─ Phase 3: 부대 교체 및 회복 전술
    ↓
Mid-Boss (선택)
    ↓
Exploration
    ├─ NPC 만남
    ├─ 아이템 수집
    └─ 사이드 미션
    ↓
Final Battle
    ├─ 챕터 보스 격퇴
    └─ 경험치 획득
    ↓
Chapter End
    ├─ 스토리 진전
    └─ 다음 챕터로
```
