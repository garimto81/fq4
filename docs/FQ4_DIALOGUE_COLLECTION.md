## 미해독 부분

### 미매핑 문자 (□)

복호화 과정에서 매핑되지 않은 문자들은 □로 표시됩니다:

- **등장 위치**: 메시지 #002, #003, #004 등 초반부에서 특히 많음
- **원인**: 저빈도 고유명사, 희귀 한자, 특수 기호
- **추정 대상**: 캐릭터명, 지명, 게임 고유 용어

**예시**:
```
[002] たぬけ□がにけさ□にのこでのすなささのこよなたぐ...
                ↑                ↑
            미매핑 문자들 - 아마도 캐릭터명이나 고유명사
```

### 특수 패턴

#### [LINE 패턴]
```
[004] うらな□がえのも□に[LINEがげきま□にのの...
                          ↑
                    대사 구분 마커로 추정
```

**의미**: 대사 줄 바꿈을 나타내는 제어 코드로 추정

#### [BREAK 패턴]
```
[051] うらな□がすいわき[BREAKがのらこ□さのるねらされじの...
                       ↑
                    일시 정지 마커로 추정
```

**의미**: 게임 연출 중 일시 정지(BREAK)를 나타내는 제어 코드

#### [PAUSE 패턴]
```
[038] たぬけ□が：[PAUSEが：ねへさき□に
                   ↑
              일시 정지 마커로 추정
```

**의미**: 게임 내 일시 정지/대기 신호

#### [CM 패턴]
```
[057] ...の[CMに8がこくのたせのばばくなさのぱなの...
            ↑
      커맨드나 제어 신호로 추정
```

## 복호화 방법론 요약

### 빈도 분석 방법

1. **Shift-JIS 바이트 추출**
   - FQ4MES 파일에서 800개 메시지 오프셋 추출
   - 각 메시지의 암호화된 바이트 시퀀스 분석

2. **빈도표 생성**
   - 일본어 일반적 텍스트의 히라가나 빈도 분포와 비교
   - 최고빈도 순으로 정렬

3. **매핑 추론**
   - 암호문 최고빈도 → 평문 최고빈도 일본어 문자로 추정
   - Tier 1부터 Tier 8까지 계층적 매핑 진행

4. **검증 및 개선**
   - 복호화된 메시지의 일본어 문법 검토
   - 조사(の, が, に, を) 패턴 확인
   - 기존 번역 자료(한글패치) 참고

### 한계점

- **Known-plaintext 부재**: 알려진 평문 없이 순수 통계 분석만 수행
- **고유명사 미해독**: 저빈도 캐릭터명, 지명은 □로 표시
- **문맥 해석**: 복호화된 텍스트의 정확한 의미는 게임 플레이로 검증 필요
